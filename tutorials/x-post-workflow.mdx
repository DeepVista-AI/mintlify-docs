## Overview

By the end, DeepVista will research, draft, and hold X posts for your review, then publish them for you — from a single prompt, with no manual scheduling.

DeepVista can turn one well-structured prompt into a complete X posting workflow: it researches source content, drafts post variants in your brand voice, holds them for your review, and publishes to your chosen channel. This tutorial walks through generating the workflow, adding a step, and publishing to X with the [Buffer CLI](https://cli.deepvista.ai).

## Prerequisites

Before you start, make sure you have:

1. A DeepVista account with an active project
2. A source of content to draw from (a URL, newsletter, or content type)
3. The Buffer CLI installed and authenticated, with your X (Twitter) channel connected in Buffer

## Step-by-step guide

<Steps>
  <Step title="Generate the workflow with one prompt">
    Paste the following prompt into DeepVista chat:

    ```text
    Create an X posting workflow for my company.
    ```

    <Tip>
      Add a line defining the themes you want the workflow to prioritize, such as AI/ML, fundraising, GTM, or technical deep-dives. This sharpens what the research step surfaces.
    </Tip>

    DeepVista generates the full workflow from this prompt. Every draft pauses for your review before anything is published.

    DeepVista automatically searches your company context to create a personalized workflow.

    <Frame>
      ![DeepVista generating a workflow from a single prompt](/images/x-post-workflow/generate-workflow-prompt.png)
    </Frame>

    The workflow appears in the chat output. Click it to open the full workflow in the right panel.

    <Frame>
      ![Workflow card in the chat output](/images/x-post-workflow/workflow-chat-output.png)
    </Frame>

    <Frame>
      ![Full workflow displayed in the right panel](/images/x-post-workflow/workflow-right-panel.png)
    </Frame>
  </Step>
  <Step title="Add a step in plain language">
    You can extend the workflow at any time. Open chat and describe the new step using this pattern:

    ```text
    Add a step after [existing step] that [does what].
    ```

    For example:

    ```text
    Add a step after "Draft review" that publishes the approved
    draft to X using the Buffer CLI.
    ```

    DeepVista handles the technical wiring. You only describe the step.

    <Frame>
      ![Adding a workflow step in plain language](/images/x-post-workflow/add-step-plain-language.png)
    </Frame>
  </Step>
  <Step title="Post to X with the Buffer CLI">
    First, find the channel ID for your connected X account:

    ```bash
    buffer channels list --fields 'items.{id,service,name}' --output json
    ```

    <Frame>
      ![Listing Buffer channels to find the X channel ID](/images/x-post-workflow/buffer-channels-list.png)
    </Frame>

    Then tell DeepVista which channel to publish to. Tell DeepVista:

    ```text
    Publish approved drafts to X using the Buffer CLI, channel ID:
    [paste your channel ID]
    ```

    DeepVista wires the publish step to build a Buffer payload and validate it before sending:

    ```bash
    buffer posts create --json "$payload" --dry-run
    ```

    <Frame>
      ![Dry-running the Buffer post payload before publishing](/images/x-post-workflow/buffer-posts-create-dry-run.png)
    </Frame>

    <Tip>
      `--dry-run` validates the payload with no API call — DeepVista runs this check on every draft before the real publish, so a malformed post never reaches X.
    </Tip>

    Once the dry run passes, the workflow sends the real request:

    ```bash
    buffer posts create --json "$payload" --output json
    ```

    <Frame>
      ![Buffer CLI publishing the approved draft to X](/images/x-post-workflow/buffer-posts-create.png)
    </Frame>
  </Step>
  <Step title="Run the workflow">
    Click the **Run** button to trigger the workflow.

    <Frame>
      ![Run button on the workflow panel](/images/x-post-workflow/run-button.png)
    </Frame>

    DeepVista populates a predefined prompt that starts the run.

    <Frame>
      ![Predefined prompt that triggers the workflow](/images/x-post-workflow/run-predefined-prompt.png)
    </Frame>

    <Frame>
      ![Workflow run in progress](/images/x-post-workflow/run-in-progress.png)
    </Frame>

    <Frame>
      ![Workflow run output with draft for review](/images/x-post-workflow/run-output.png)
    </Frame>

    Once you approve the draft, the workflow publishes it to X via the Buffer CLI.
  </Step>
</Steps>
